Homoglyph Attacks

Unicode look-alike characters used for phishing and impersonation

What is a Homoglyph Attack?

A homoglyph attack exploits characters that look identical but belong to different writing systems. Unicode contains thousands of such characters across Latin, Cyrillic, Greek, Armenian and many other scripts.

By substituting characters with visually identical ones, attackers can create identifiers that appear legitimate to humans while being completely different strings to computers.

Simple Example

paypal.com раураl.com

The second line uses Cyrillic characters. Most people will read both lines as “paypal”. To software they are entirely different domain names.

Real World Attacks

IDN phishing domains

Internationalized Domain Names allow Unicode characters in domains. Attackers register domains visually identical to popular websites and use them for phishing pages that collect passwords or financial data.

Apple login spoof (2017)

Security researchers demonstrated a fake apple.com domain created with Cyrillic letters. Some browsers displayed the Unicode form instead of punycode, making the malicious site indistinguishable from the real one.

Open-source package impersonation

Malicious packages have been uploaded to package registries using names that visually match legitimate libraries. Developers installing dependencies can easily miss the subtle differences.
